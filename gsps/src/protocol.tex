\begin{abstract}
The Geometric Semantic Positioning System (GSPS) is a framework designed to project high-dimensional semantic vectors into deterministic geometric coordinates. The protocol's strength lies in its simplicity: it utilizes relative semantic similarity to dynamically position concepts within bounded spaces. This framework-agnostic approach ensures that any system sharing the same anchor configuration will compute identical spatial coordinates for a given concept, independent of a shared database.
\end{abstract}

\section{Core Mechanics}

\subsection{Anchor-Defined Spaces}

Semantic space is bounded by 1--4 fixed anchor nodes positioned at predetermined coordinates. These serve as the absolute ground truth for the layout's coordinate system:

\begin{itemize}
    \item \textbf{4-Anchor:} Square configuration $[(-1, 0), (1, 0), (0, 1), (0, -1)]$
    \item \textbf{3-Anchor:} Triangle configuration $[(-1, 0), (1, 0), (0, 1)]$
    \item \textbf{2-Anchor:} Edge configuration $[(-1, 0), (1, 0)]$
    \item \textbf{1-Anchor:} Point configuration $[(0, 0)]$
\end{itemize}

\subsection{The Equilibrium Engine}

For spaces defined by two or more anchors, inner nodes are positioned using an auto-balancing push-and-pull system based on cosine similarity.

\paragraph{1. The Dynamic Pivot}
Instead of relying on absolute similarity thresholds, GSPS calculates a ``dynamic pivot'' for each incoming concept. This pivot is the average cosine similarity between the concept and all the fixed anchors in the layout.

\paragraph{2. Relative Forces}
The system then compares the concept's similarity to each specific anchor against that baseline pivot:
\begin{itemize}
    \item \textbf{Attraction (Gravity):} If the similarity to an anchor is higher than the pivot, that anchor pulls the node toward it.
    \item \textbf{Repulsion:} If the similarity is lower than the pivot, the anchor pushes the node away.
\end{itemize}
The strength of this force scales naturally with how far the score deviates from the pivot.
